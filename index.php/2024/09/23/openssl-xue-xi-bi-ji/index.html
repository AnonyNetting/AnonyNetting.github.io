    OpenSSL学习笔记

    发布于 4 分钟前  4 次阅读

    做关于PQ-TLS的硬件相关调查时发现如果要让成果融入现有基础设施，那么大概一定要在软件接口上对接OpenSSL。

    以Intel Quick Assist Technology为例："QAT Engine 是QAT技术方案的核心模块，主要的作用是作为应用程序和硬件之间的中间层，负责“加解密操作的输入输出数据”在用户应用程序与硬件卡之间进行传递，主要操作就是IO的读写。QAT Engine 是以 OpenSSL 第三方插件的方式提供给用户，意味用户可以使用 OpenSSL 标准的API，就可以实现对TLS的加速，只需要对原有代码做OpenSSL异步改造，就可以享受QAT技术带来的TLS性能加速，业务侵入性较小。"

    如上图所示，QAT  Engine 支持两种加速方式：

    • 软件加速（qat_sw）：使用 Multi-Buffer （SIMD）技术，对密码算法进行并行处理优化。
    • 硬件加速（qat_hw）：使用QAT硬件加速卡，将密码算法计算从CPU OffLoad到硬件加速卡。

    以上引自: TLS 加速技术：Intel QuickAssist Technology（QAT）解决方案

    无论以SIMD还是offload方法，首先要解决如何对接OpenSSL的问题。以下内容根据OpenSSL 中文手册来总结OpenSSL相关知识。

    1基础知识

    对称算法

    简单来说就是加密密钥和解密密钥相同的算法。重点在于由于密钥是有限长而原文是不定长，如何对原文执行分段就成了问题：

    1. Electronic Code Book(ECB)
    2. Cipher Block Chaining(CBC)
    3. Cipher Feedback Mode(CFB)
    4. Output Feedback Mode(OFB)

    简单来说，ECB每块分段采用相同密钥和各自原文作为输入；CBC的(n+1)段采用相同密钥，(n)段密文与(n+1)段明文的异或共同作为输入，对于第1段，其0段密文不存在，所以需要设置一个初始化向量代替0段密文；CFB和OFB将分段做得更精细，大致思路没有太大变化。

    摘要算法/公钥算法

    略过。

    回调函数

    看起来似乎就是函数指针。OpenSSL允许用户自定一些底层功能的实现方法，比如假定用户调用OpenSSL的某个内部需要生成随机数（用户提供的生成随机数函数称为B，OpenSSL默认提供的随机数函数称为C）的函数（OpenSSL提供的这个上层函数称为A），那么在调用A时就可以通过提供B的函数指针来指定随机数生成方法；如果不指定，则使用OpenSSL提供的默认C。

    2OpenSSL简介

    OpenSSL是用C开发的开源工具，功能包含：SSL协议实现(包括SSLv2、SSLv3和TLSv1)、大量软算法(对称/非对称/摘要)、大数运算、非对称算法密钥生成、ASN.1编解码库、证书请求(PKCS10)编解码、数字证书编解码、CRL编解码、OCSP协议、数字证书验证、PKCS7标准实现和PKCS12个人数字证书格式实现等功能。

    安装跳过，参考原文即可。

    OpenSSL源码

    主要包括eay库、ssl库、工具源码、范例源码以及测试源码。eay库主要包含具体功能，其中重点留意硬件引擎(crypto/engine目录)，硬件引擎接口：用户如果要写自己的硬件引擎，必须实现它所规定的接口；ssl库包含ssl, tls, dtls等ssl协议的源码，各个版本基本上都有客户端源码(*_clnt.c)、服务源码(*_srvr.c)、通用源码(*_both.c)、底层包源码（*_pkt.c）、方法源码(*_meth.c)以及协议相关的各种密钥计算源码(*_enc.c)等。
