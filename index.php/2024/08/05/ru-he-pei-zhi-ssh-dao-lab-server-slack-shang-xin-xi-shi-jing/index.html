    如何配置ssh到Lab Server？Slack上信息释经

    发布于 16 天前  23 次阅读

    本文致力于完整搜集解释Slack上所有与ssh配置有关的信息，并给出一份尽可能详细且易懂的ssh到Lab Server的指南。忘记你的tutor，照着这篇文章做，你自己也能配置好Lab的网络环境！

    1.生成配对公钥和私钥作为你的基本身份认证

    首先保证你的system user name不含特殊字符(包括中文，日文等，否则后续生成密钥可能出现识别问题)。打开你的Powershell程序(windows环境，下同；linux和mac请自行尝试确认具有相同功能的程序或路径)，输入

    ssh-keygen -t ecdsa -b 521

    随后保持默认即可，不需要输入passphrase(也可以输入，这主要是用来作为keygen的额外参数用来成ssh的public key和secret key)。以上指令在你的C:\Users\<YourUserName>\.ssh文件夹生成了两个密钥文件，分别是id_ecdsa和id_ecdsa.pub。将id_ecdsa.pub，和一个你自行决定的user name发送给教授，让他将你的公钥上传到服务器上，并为你创建一个user account。同时教授应该会告知你的user account的初始密码，如果没有，询问他。

    2.连接到Lab Server并调整初期设置

    在Lab以外网络的条件下(如果你在Lab，使用UTokyo-WiFi，完成Step2后可以重新使用Lab网络)在powershell中输入ssh <UserName>@<alice_addr>，(<alice_addr>替换为alice的地址，在此不便写出，另外最后的.93应当改成.193)。返回结果应当提示输入密码，密码是教授在Step1最后返回的初始密码，以及询问是否本机存储这次连接写入knownhosts，确认存入。

    在左侧已经显示[UserName@alice ~]的条件下输入passwd，确认当前password后修改为你的自定密码。

    在左侧已经显示[UserName@alice ~]的条件下输入ssh 192.168.11.2，左侧应当变更为[UserName@emperor ~]。在左侧显示变更后的条件下输入vncserver，返回结果New 'emperor:<number> (<UserName>)' desktop is emperor:<number>。记住这个<number>。随后继续输入vncpasswd，自定一个vncserver的密码。

    3.配置你的ssh config文件(可选)

    我们在C:\Users\<YourUserName>\.ssh文件夹创建一个config文件，在这个文件中写入以下内容:(删除所有注释)

    Host <A> // <A>替换为你在Lab网络以外连接到Lab服务器时所指定的名称
    HostName <address> // <address>替换为alice的地址，在此不便写出，另外最后的.93应当改成.193
    User <UserName>//<UserName>替换为你在Step1中发送给教授的user name
    IdentityFile ~/.ssh/id_ecdsa
    LocalForward 8000 192.168.11.2:59<vnc_addr> /* 8000可以替换为其他数字，这一行的功能表示你的本地端口8000与192.168.11.2的<vnc_addr>建立连接；<vnc_addr>替换为步骤2中我们得到的<number>，与59之间没有空格 */
Host <B> // <B>替换为
    HostName 192.168.11.1
    User <UserName>//<UserName>替换为你在Step1中发送给教授的user name
    IdentityFile ~/.ssh/id_ecdsa
    LocalForward 8000 192.168.11.2:59<vnc_addr> /* 8000可以替换为其他数字，这一行的功能表示你的本地端口8000与192.168.11.2的<vnc_addr>建立连接；<vnc_addr>替换为步骤2中我们得到的<number>，与59之间没有空格 */

    当我们完成这些配置后，连接到服务器的指令就可以简化为

    ssh <A> // <A>替换为你在Lab网络以外连接到Lab服务器时所指定的名称
ssh <B> // <B>替换为你在Lab网络内连接到Lab服务器时所指定的名称

    4.vnc可视化界面配置

    在vncviewer创建一个新的connection，如果执行了Step3，那么新建Connection的Properties的VNC Server处输入在步骤3中指定的本地端口，例如指定8000时，此处配置localhost:8000。随后在powershell界面退出alice和emperor，通过ssh <A>或ssh <B>(根据网络连接位置决定)重新连接到alice后，在VNCViewer启动刚才建立的connection，初次启动时需要Step2中设置的vncpasswd。随后进入可视化界面，输入Step2中通过passwd命令设置的自定密码，完成全部流程。

    如果没有操作step3，那么在VNC Server处输入任意1024-65535之间的端口号<local_port>（实际上根据本机情况可能部分端口不可用）。随后在powershell界面退出与alice和emperor，通过ssh <UserName>@<alice> -L <local_port>:192.168.11.2:59<number>指令也可以建立本地端口与server端口的连接。随后在VNCViewer启动刚才建立的connection，初次启动时需要Step2中设置的vncpasswd。随后进入可视化界面，输入Step2中通过passwd命令设置的自定密码，完成全部流程。
